# WatchGuard AEPDR

CIS Controls v8 safeguard mappings for **WatchGuard AEPDR**.

| CIS Safeguard | Safeguard Title | Alignment | Evidence |
|---|---|---|---|
| 1.1 | Establish and Maintain a Detailed Enterprise Asset Inventory | Facilitates | WatchGuard Endpoint Security solutions include an Inventory facility for hardware across managed devices and with details per device: CPU, RAM, storage, TPM, BIOS information, system type, virtual or physical, OS details, private and public IP and MAC addresses. This can assist with maintaining an Enterprise Asset Inventory. |
| 1.2 | Address Unauthorized Assets | Full | WatchGuard EDR/EPDR/AEPDR include Device Isolation capabilities; geolocation, lock and remote wipe are also available for mobile managed devices. If a managed device becomes unauthorized (loss or theft, e.g.), it will be possible to take action accordingly. |
| 1.3 | Utilize an Active Discovery Tool | Full | WatchGuard Endpoint Security includes a Discovery feature to find unmanaged computers and devices on the network (on-demand or periodically) and optionally remotely deploy the WatchGuard Endpoint Agent to supported devices that do not have it installed. |
| 2.1 | Establish and Maintain a Software Inventory | Facilitates | WatchGuard Endpoint Security includes a software Inventory facility, detailing installed software across managed devices and per device: Name, Publisher and Version. It is possible to check, for each software package detected, the number of devices that have it installed, as well as monitor software installations and uninstallations in each device. |
| 2.4 | Utilize Automated Software Inventory Tools | Partial | WatchGuard Endpoint Security updates its Software Inventory automatically every hour. |
| 2.5 | Allowlist Authorized Software | Partial | WatchGuard AEPDR includes the Zero-Trust Application Service that classifies processes as either malware or as trusted before letting only the trusted execute on each endpoint, thereby enabling the ultimate default-deny posture. Any installation of software is precided by a classification of that installer, that will be allowed only if it is trusted by WatchGuard. Cloud-based applications are outside the scope of this Service. |
| 2.6 | Allowlist Authorized Libraries | Full | WatchGuard AEPDR includes the Zero-Trust Application Service that classifies processes as either malware or as trusted before letting only the trusted execute on each endpoint, thereby enabling the ultimate default-deny posture. Any installation of software is precided by a classification of that installer, that will be allowed only if it is trusted by WatchGuard. |
| 2.7 | Allowlist Authorized Scripts | Partial | WatchGuard Advanced AEPDR includes Advanced Security Policies to Block (and Audit) any scripts that have not yet been classified by WatchGuard’s Security Intelligence, in addition to Powershell scripts with obfuscated parameters or run by interactive user accounts. |
| 4.1 | Establish and Maintain a Secure Configuration Process | Facilitates | WatchGuard Endpoint Security Solutions provide default endpoint security settings that cannot be modified. Different settings can be established in separate profiles. A dedicated Risk Monitoring dashboard facility provides a health overview for the implementation, identifies security misconfigurations, unprotected endpoints, unaddressed Indicators of Attack and security patches pending deployment. The WatchGuard Endpoint Security API allows retrieving detected risk information and details, as well as updating risk configuration. |

Program rules : Allow or prevent the programs installed on users’ devices from communicating with other devices. Intrusion detection system: Detects and rejects malformed traffic patterns that can affect the security or performance of protected devices. |
